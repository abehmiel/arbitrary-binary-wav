# Design: uv Migration & Pythonic Modernization

**Date:** 2026-03-29
**Status:** Approved

## Summary

Migrate `arbitrary-binary-wav` from Poetry to uv, replace the `click` CLI with `argparse`, rename the script to a proper module with an entry point, and add ruff linting. The runtime logic is unchanged.

## Project Structure

```
arbitrary_binary_wav.py   # renamed from convert.py
pyproject.toml            # rewritten for uv + hatchling
uv.lock                   # generated by uv sync
README.md                 # update install/usage instructions
```

`poetry.lock` is deleted.

## pyproject.toml

- Build backend: `hatchling`
- Python floor: `>=3.11`
- Dependencies: `scipy>=1`, `numpy>=1` (drop `click`, drop `wave` — stdlib)
- Entry point: `arbitrary-binary-wav = "arbitrary_binary_wav:main"`
- Ruff config: `target-version = "py311"`, lint rules `["E", "F", "I", "UP"]`

## arbitrary_binary_wav.py

Changes from `convert.py`:
1. Replace `click` decorators with `argparse.ArgumentParser` in a `main()` function
2. Wrap `wave.open(...)` in a `with` block (fixes manual `.close()`)
3. Rename `output_sound` → `main` (entry point function)
4. `output_wave()` pure logic function is unchanged
